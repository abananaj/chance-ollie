2. Key Technical Concepts:
   - **`query_loop_block_query_vars`** WordPress filter hook — the correct hook (over `pre_get_posts`) for modifying `core/query` block queries; receives `($query, $block, $page)` where `$block->attributes` contains the query block's saved attributes
   - **`register_block_type_args`** PHP filter — registers custom attributes server-side on `core/query` so they survive block validation and are available in `$block->attributes` at render time
   - **`addFilter('blocks.registerBlockType')`** JS — registers custom attributes client-side so they appear in the block's saved markup
   - **`addFilter('editor.BlockEdit')`** + **`createHigherOrderComponent`** — wraps BlockEdit to inject InspectorControls panels for the `core/query` block
   - **IIFE pattern** for block editor JS — uses `wp.*` globals (no ES imports, no Vite bundling), stored in `inc/metadata/js/` (web-accessible as a theme subdirectory), enqueued via `enqueue_block_editor_assets` with WordPress script dependencies
   - **WP_Meta_Query types**: CHAR (string), DECIMAL (number), NUMERIC (boolean), DATE/DATETIME/TIME (date-time, auto-detected after normalization)
   - **WP_Meta_Query compare operators**: `=`, `!=`, `>`, `>=`, `<`, `<=`, `LIKE`, `NOT LIKE`, `EXISTS`, `NOT EXISTS` — whitelisted per type for security
   - **ACF date_picker storage**: uses Ymd format (e.g. `20261203`) by default — YYYYMMDD strings sort correctly lexicographically so `type => 'CHAR'` comparison is valid
   - **Date normalization**: handles Unix timestamps (10-digit), compact YYYYMMDDHHmmss (14-digit), compact YYYYMMDD (8-digit, ACF default), and standard SQL formats
   - **`enqueue_block_editor_assets`** action — used to enqueue editor-only JS with `filemtime()` cache-busting
